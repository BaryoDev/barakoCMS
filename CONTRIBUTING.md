# Contributing to BarakoCMS

We love your input! We want to make contributing to BarakoCMS as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## "Vibe Coding" & AI Agents

We explicitly welcome contributions generated by AI agents (Cursor, Windsurf, etc.).
- Please ensure your AI follows the `.cursorrules` defined in the root.
- Mention in your PR if the code was AI-generated (it helps us review!).

## Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!
